# 写在前面



你好，我是大圣



当你跟扣子对话的时候，如果每次对话都从零开始，扣子就是一个每天失忆的员工。



你每次都得重新自我介绍、重新交代背景、重新解释你的项目。这不叫协作，这叫重复劳动



扣子用三个模块解决了这个问题：记忆、文件、收藏夹



这节课就把这三个模块的能力和边界讲清楚



# 一、先记住一句话



**记忆是扣子自己记的，文件是双方的产物库，收藏夹是你主动喂的**



这句话是这节课的全部。后面所有内容都是在拆解这句话



![](https://feishu.cn/file/UXZwbRR3YobsaMx335FctqYHnOf)



# 二、记忆：扣子自己记的笔记本



## 2.1 它记什么



扣子会从你们的对话中自动提取它认为重要的信息，存成 Markdown 文件



你不需要手动操作，它自己判断、自己记



它记的东西包括：你是谁（姓名、职位、行业）、你的偏好（喜欢什么风格、讨厌什么表达方式）、你正在做的项目、你交代过的重要事实（"新品推广截止日期是3月31日"）、你们之间的协作约定



**这些信息不是存在模型脑子里的，而是存成了具体的文件**。



扣子为每类信息准备了专门的档案：



- **SOUL.md**：扣子自己的角色设定，它叫什么、什么性格、怎么说话
- **USER.md**：关于你的画像，你的身份、偏好、称呼方式
- **MEMORY.md**：长期记忆库，跨对话的关键事实和结论
- **TOOLS.md**：工具使用记录，常用的工具和调用习惯
- **SECRET.md**：隐私信息，你授权给它的账号、密钥



## 2.2 它怎么记



两种方式并行：



**自动提取**：对话结束后，扣子会在后台静默整理，把值得长期保留的信息写入对应的文件。你不需要做任何事



**你主动指定**：遇到特别重要的信息，你可以直接告诉它：我在公司是产品运营，请重点记住我的身份角色



## 2.3 它怎么用记忆



扣子会自动检索记忆，这个不需要你指定



这也是为什么扣子强调越用越好用，不是说模型变得有多强，而是你积累的记忆越变越多，它对你的理解变深了



## 2.4 记忆的两个重要特性



**跨渠道共享**：你在网页端告诉扣子的事，在飞书里问它也记得。记忆是全局的



**用户隔离**：如果你把扣子拉进飞书群，其他人在群里问，扣子不会泄露你在私聊里说过的信息。这是硬性隔离



## 2.5 记忆出了问题怎么办



你可以打开文件系统里的记忆目录，直接查看每个 Memory.md 的内容



如果发现记错了或者遗漏了，在对话里告诉它纠正就行：你记错了，我不是负责市场的，我是负责运营的



# 三、文件：双方的产物库



文件系统的定位很简单：**存东西的地方**



东西从两个方向来：



**你上传的**：图片、文档、表格、PDF、音视频



你在对话中上传的任何素材，都会按对话主题自动归档到文件目录里



**扣子产出的**：它帮你做的PPT、生成的报告、分析的图表、制作的视频



所有产物也会自动存进来



## 3.1 文件能做什么



**引用到对话**：你可以从文件列表里选一个文件，点"引用到对话"，然后基于这个文件发起新任务。不需要重新上传



**在线编辑**：部分格式（Excel、PPT、drawio）可以直接在扣子里在线编辑。每次改动自动保存版本，改错了可以回退



**下载和分享**：产出的文件可以下载到本地，也可以生成分享链接给别人



## 3.2 文件和记忆的区别



这是最容易混淆的地方



**记忆**存的是认知：扣子对你这个人的理解。它不存具体的文件内容，只存关键信息的摘要



**文件**存的是实体：具体的文档、图片、视频、表格。它不理解内容，只负责归档和调取



打个比方：



**你的员工记住了老板喜欢简约风格的PPT，这是记忆**



**他把做好的PPT存在了公司网盘里，这是文件**



# 四、收藏夹：你主动喂的知识库



**收藏夹是你有意识地、主动地往扣子脑子里灌输知识**



## 4.1 它的运作方式



1. **建标签**：比如你要持续关注英伟达，就建一个"英伟达"的标签。标签是收藏夹的分类单位



1. **收藏内容**：日常看到好的文章、报告、截图，往这个标签里扔。支持粘贴文本、链接、上传附件、从微信导入



1. **对话中@调用**：需要的时候，在对话框里输入@一人公司，扣子会把这个标签下所有内容加载进上下文，然后基于这些知识来回答你的问题或执行任务



## 4.2 收藏夹的核心场景



**行业热点追踪**：把日常看到的行业资讯、大佬观点、研报截图扔进对应标签。需要的时候一键@，让扣子帮你汇总、分析、对比



**工作资产库**：品牌介绍、产品卖点、常用模板、历史方案统一归档到标签下。任何需要对齐口径的场景，@标签就行



**知识沉淀**：不只是收藏外部内容，扣子给你产出的好东西也可以反向收藏。越用越厚的知识库



## 4.3 我个人的一个看法



我其实不太喜欢这种标签的设计，因为它让页面看起来很杂乱



我更喜欢的是创建一个文件夹，然后在这个文件夹下去加入文档



在我们和扣子对话的时候，可以直接@文件夹



当然，这只是我现在的想法，也许用着用着会发现它的标签可能有它的道理

# 五、实操建议：什么信息该放哪



![](https://feishu.cn/file/JyJEbtXYDodNDtxJPEVcGYgRnWY)



**关于你的信息**（身份、偏好、项目、习惯）→ 让扣子自己记就行，特别重要的可以主动强调"重点记住：……"



**工作中的产物**（PPT、报告、图表、你上传的文件）→ 不用管，文件系统自动归档



**外部知识**（行业资讯、竞品信息、品牌资料、参考文章）→ 手动收藏到收藏夹，打好标签，用的时候@调出来



# 写在最后



我希望大家通过这节课，不仅学习的是扣子的记忆系统



大多数的agent系统都是类似于这样的一个体系



后面不管你是切换到其他的平台，还是未来有新的工具出现，大家的设计理念都是类似的



另外你在用扣子的记忆的时候，你可能会发现它没那么好用



这个大家一定要有预期，AI还在一个快速发展迭代的阶段，它现在没有大家想的那么强
