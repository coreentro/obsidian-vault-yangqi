# 写在前面



你好，我是大圣



这节课跟上一节课会有一些重合，但我要再次强调一下，因为它决定了我们学扣子到底要学哪几个模块



扣子现在的产品线已经不是一个单一工具了



咱们先不讲操作细节，只讲结构关系。搞清楚我现在站在哪，后面每一步才不会迷路



**PS：在写这节课的时候，2026年4月7号，扣子发布了2.5版本**



# 一、扣子的三块地盘



地址：https://www.coze.cn/



下面这张图不是扣子所有的内容，但它是我们核心需要学习的内容



![](https://feishu.cn/file/QiJbbTO1Mo7vcxx5Kk3cEurHn4d)



我用一个比喻来让大家去记住这三者的关系：



- **扣子** = 前台干活的人
- **扣子编程** = 后台造能力的工厂
- **工作流编排** = 工厂里上一代的流水线（**还在运转，但不再是主角**）



## 1）扣子：直接帮你干活



这是扣子的主界面



打开扣子 APP 或网页，面对的就是它。你提需求，它完成任务



写报告、搜资料、做 PPT、分析数据，所有"用 AI"的体验都在这里发生



它的能力上限取决于一个东西：**它装了多少技能（Skills）**



技能越多，能做的事越多。没有技能的扣子，和一个空白手机没有区别



## 2）扣子编程 ：给 Agent 造技能的地方



如果扣子是手机，扣子编程就是 APP 工厂



你在这里做三件事：



1. 开发技能（Skills）：一个技能就是一个可复用的能力模块
2. 搭建智能体（Agent）：组合多个技能，形成一个完整的 AI 助手
3. 构建应用：网站、小程序、API 接口，都能在这里一键部署



现在的扣子编程已经升级成了 Vibe Coding 平台。



你不需要会写代码，用对话就能搭，搭完一键云端部署



## 3）工作流编排：上一代的造法



2024 年的扣子，核心玩法就是拖拽工作流：你手动定义每一步流程，AI 按你画好的路线执行



这种方式现在还保留在扣子编程里，但已经从唯一的变成了其中的一种



现在你有三种造法：



- Vibe Coding（对话式编程，口喷需求）
- 工作流编排（拖拽式，手动搭建）
- 两者结合



**为什么还要提它？** 三个原因：



第一：这是扣子的来时路，你需要知道；而且我们也有一套完整的体系化的课程



第二：有些场景，工作流编排是最稳的方案；因为他能让你看得见，摸得着



第三：市场上对智能体的很多认知还停留在工作流时代，如果你有对应的业务，那学习工作流编排会有帮助



# 二、三者的关系



我再用一个例子来类比三者的关系：



**扣子 = 手机**，它本身就能干活，但装的 APP 越多，能做的事越多



**扣子编程 = APP 工厂**，批量生产各种 APP（技能、智能体、应用），装到手机里



**工作流编排 = 工厂里的老设备**，还能用，但新产线（Vibe Coding）已经更快了



一个关键的路径：**扣子编程 → 产出技能 → 装到扣子上 → 扣子变强**



而且你不一定要自己造技能



技能商店里有别人做好的，直接装上就行。但如果你想深度定制，想卖 AI，扣子编程就是你的主阵地
