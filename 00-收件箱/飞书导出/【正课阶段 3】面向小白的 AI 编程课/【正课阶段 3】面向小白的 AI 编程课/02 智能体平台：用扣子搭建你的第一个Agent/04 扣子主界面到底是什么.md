# 写在前面



你好，我是大圣



这节课我们来讲扣子，它的主界面到底是什么？它背后的产品逻辑是什么？



# 一、一句话理解扣子主界面



**扣子主界面 = 一个能干活的 Agent 的完整工作环境**



跟普通的chatbot不同，扣子背后是一套让AI从被动回答变成主动干活的基础设施



理解扣子的主界面，你可以类比招一个新员工



新员工来了，你需要给他什么？



- 你需要跟他沟通，告诉他做什么；你需要他能记住事情，别每次都重新交代；



- 你需要给他配电脑、配工牌、配邮箱；你需要给他培训，让他掌握专业技能；



- 然后他才能产出成果写报告、做方案、剪视频



扣子主界面做的事情，就是把这五件事全部搭好了：



![](https://feishu.cn/file/U9clbQS1qoRPTKxmbh3cnVKgnVe)



# 二、逐层拆解：每一层在干什么



## 第1层：沟通：对话系统



这是你和扣子之间唯一的交互方式。你不需要点按钮、不需要填表单，**所有操作都通过自然语言完成**



帮我做个PPT是对话；每天早上10点给我发AI日报也是对话；用云手机帮我看看微博热搜还是对话



对话不只是聊天，它是扣子的操作系统。你发出的每一句话，本质上都是一条指令



扣子还有一个重要的设计：**前台任务和后台任务的分离**



简单的事它当场做完。复杂的事：做视频、操作云手机、生成网页，它会转到后台跑，不占你的对话窗口。



你可以一边聊天一边等它出结果。这意味着你可以同时给它派多个任务



## 第2层：记忆：它的大脑



如果只有对话没有记忆，扣子就是一个每天失忆的员工



你每次都得重新自我介绍，重新交代背景



扣子的记忆系统由三个模块组成，后面会专门用一节课讲清楚它们的区别。



这里先记住它们各自的角色：



**记忆（Memory）**：扣子自动记的笔记本。它会从对话中提取关键信息：



你是谁、你的偏好、你在做什么项目，存成 Markdown 文件。你不需要手动操作，它自己记



**文件系统**：扣子的云盘。你上传的资料存在这里，扣子产出的报告、图表、PPT也自动存在这里



**收藏夹**：你主动喂给扣子的知识库。行业报告、竞品截图、品牌资料，你收藏进去，打上标签，对话时用@调用



三者的关系：记忆是扣子自己记的，文件是双方的产物库，收藏夹是你主动喂的。



各管各的，但在对话中可以同时调用



## 第3层：装备：它的手脚和身份



前两层解决的是你和扣子一起工作。第3层解决的是另一个问题：**你不在的时候，扣子能不能自己干活？**



答案是能，但它需要三样东西：



**日程**：扣子的排班表。你告诉它"每天早上10点收集AI行业动态"，它就会自己创建一个循环任务，到点自动执行，做完给你发结果。你不需要盯着，它自己跑



**云设备**：扣子的手和眼。云手机让它能下载APP、刷小红书、查大众点评。



云电脑让它能跑代码、浏览网页、装软件。



这两样东西让扣子突破了只能在聊天框里干活的限制，真正进入了物理世界的应用层



**邮箱**：扣子的身份证。有了邮箱，它可以注册第三方平台的账号、和别人通信、和其他Agent交朋友。这是它在数字世界里拥有独立人格的起点



三者合在一起，回答的是同一个产品设计问题：



**怎么让AI从被动响应变成主动执行？** 



日程让它知道什么时候干，设备让它有手去干，邮箱让它有身份去干



## 第4层：技能：它的能力上限



这是扣子主界面里最重要的一层设计



一个没有技能的扣子，就像一个刚入职的新员工，聪明但什么都不会。你让它做事，它只能靠通用能力硬撑



给它装上技能之后，它就变成了一个有专业经验的老员工。**技能本质上是可复用的任务解决方案**



技能有四个来源：



**官方内置的**：PPT、Excel、视频、海报、播客，这些是扣子出厂就带的能力



**技能商店买的**：别人开发好的技能，法律咨询、金融分析、自媒体运营……按次付费或订阅使用



**你自己造的**：把一次满意的对话沉淀为技能，或者去扣子编程里用Vibe Coding开发



**复制改造的**：看到商店里有个开源技能不错，但不完全适合你？复制过来，改成自己的版本



技能系统的设计哲学是：**让AI从理解你的指令进化到掌握你的方法**



## 第5层：产出：它能交付什么



有了前面四层的支撑，扣子能产出的东西就非常具体了：



视频（内置Seedance 2.0，从剧本到成片）、PPT（自动大纲+排版+配图）、海报（多风格多尺寸）、播客（文本转双人对谈音频）、Excel（数据分析+图表+模板生成）、文档、图片、网页……



这些能力的调用方式是统一的：在对话框里 @对应的技能，然后描述你的需求



# 三、扣子的产品经理在想什么



我很喜欢去分析大厂的Agent产品形态，因为它背后代表着最顶级的产品经理对Agent这个产品的理解



以上这些功能，暴露了扣子产品团队的核心思路：



**他们在造的不是一个工具，而是一个数字员工的运行环境**



数字员工的逻辑是：它有记忆、有身份、有设备、有技能、有排班表。你不在的时候它也在工作。你用得越久，它越了解你



这也是为什么扣子2.0之后改了Slogan：



从AI辅助工具变成了主动协作伙伴。不是营销话术，是产品架构的真实反映
