# 写在前面



你好，我是大圣



这节课我跟你聊清楚一件事：扣子工作流的手动编排，到底还要不要学？



# 一、我与这门课的渊源



2024年6月，我们这门课的第一版上线，当时只有7节大课



2024年12月，我把它重构了一遍，7节大课拆成130多节小课，按初中高三级分层



![](https://feishu.cn/file/Y8CSblbBboEjIXxRizVcTf64ncb)



从2024年的12月份到2025年10月份，课程主体一直围绕扣子平台



但是随着AI的发展，各种产品形态层出不穷，涌现出了像腾讯的WorkBuddy、Kimi Code、Claude Code、小龙虾这样的智能体形态，扣子产品本身也进行了非常大的重构



因此，我又做了第三次重构，在2026年的4月份彻底成型



这一轮下来，以前那套130多节的扣子手动工作流编排课程从核心主线转为了辅助补充内容



**但这不代表它没有价值**  
  
**截止到现在，你去看自媒体上卖智能体课的，很多还在主推扣子工作流**



**允许我自夸一句：我们的课确实太超前了，一直在迭代，一直在跟最新的AI趋势走**



这节课我就跟你讲清楚，你到底要不要去学习那130节课，什么情况下需要学，以及怎么学？



# 二、你该如何学习扣子手动编排工作流



## 2.1 先搞清楚一个规律



越早期的知识，有一个优点，有一个缺点



优点是：**越早的东西越底层，越能暴露本质逻辑**



你如果啃下来了，后面那些封装得更完善的工具，你上手会更快，认知也更深。



就像电脑刚出来那会儿学编程，可能比较痛苦，但那个年代的编程大神到今天还是大神



缺点是：越早的东西可能已经过时了，现在有更方便的方式替代它。你如果一头扎进去，会浪费时间



所以我不想让你纠结"要不要学"，我想帮你想清楚"该怎么学"



## 2.2 我的建议：按需学习



我学AI的理念一直是：**干中学，用中学，教中学**



当你把AI通识了解完，了解了WorkBuddy，了解了扣子智能体，后面还有Kimi Code、Claude Code



课程已经把智能体最完整的形态给你讲明白了



接下来你要做的，不是继续往深处钻，而是**去找需求**



工作里的、生活里的，找到一个实际的事儿，然后用AI去解决它。这是一切的出发点



在这个过程中：



你先把我们的课程通览一遍，然后选择一个合适的工具



如果你现有的知识储备已经能很好地满足你的需求，那你就不需要往下走



但如果你发现自己对工作流、对智能体的理解还是有点模糊，脑子里的概念拎不清



OK，这个时候再去学手动编排。它更底层、更细节，能让你看到这件事的本来面目



还有一种情况：你是讲师，或者要在企业里给别人做培训



我建议你去学，因为作为讲师，你必须把一件事的来龙去脉搞得很清楚，才能给别人讲清楚



最后多说一句：**把130节手动工作流编排全部看完的伙伴，他们对AI的理解，跟别人是不一样的**



# 三、扣子工作流编排的手动入口



最后再给大家说一下这个手动编排工作流的入口



三个入口分别是右上角的回到旧版，还有下面的低代码区块的两个部分



这里我要多说一句，扣子也在不断地迭代，也许有一天这两个入口的位置会变，大家不用慌，它大概率会保存。因为在2024年到2025年，很多人的资产就是手动编排的工作流，它们被大量用在工作和生活当中



所以这部分的内容扣子不会轻易下线
