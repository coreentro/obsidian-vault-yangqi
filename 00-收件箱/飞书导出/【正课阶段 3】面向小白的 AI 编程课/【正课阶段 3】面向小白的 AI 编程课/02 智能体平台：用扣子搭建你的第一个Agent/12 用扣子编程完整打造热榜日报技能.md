# 写在前面



你好，我是大圣



这节课带大家用扣子编程做一个真实的案例



生成一份每日热榜日报页面。从 3 个热榜接口抓数据，自动生成一个漂亮的 HTML 文件，双击就能看



这节课的重点不是教你写代码。而是**你怎么把需求说清楚，让AI替你写**



# 一、我们要做什么



做一个技能，每天从 36 氪、百度、抖音抓热榜数据，自动生成一个静态 HTML 页面



![](https://feishu.cn/file/Ugh9bBLCSotzWkxCQqgcmt4Wnme)

为什么选这个案例？



因为它满足三个条件：



1. 有真实价值（每天能产出可读的内容）
2. 结构完整（抓取→清洗→生成，流程闭环）
3. 对小白友好（不需要数据库、最终就是一个 HTML 文件）



这是一个麻雀虽小，五脏俱全的案例



# 二、第一步：找到数据源



随着你用AI做越来越多的事情：**你会发现最重要的根本不是代码，而是稳定且高质量的数据源**



只要你有数据，AI几乎啥都能干（**请务必记住我这句话**）



在这个需求里面，第一步我们要做的就是：帮助AI找到合适的热榜数据接口



我找了 3 个国内可直接访问的热榜接口：



```Markdown
https://api.iyuns.com/api/hot36kr
https://api.iyuns.com/api/baiduhot
https://api.iyuns.com/api/douyinhot
```



![](https://feishu.cn/file/WItxbsoJbo8nMmxjXVXckumRnNR)



你可能会好奇AI不是能搜索吗，它为什么不能帮我找？



这句话没问题，但从需求流程上来讲，不管是AI帮你找，还是你自己百度搜索，还是你找朋友去问



最终的目的都是要找到合适的数据源，在这个教程里面，咱们就不展示所有的细节了



# 三、第二步：想清楚你要什么



所有的应用：都是输入、处理和输出



所以在AI动手做一件事之前，你需要先在脑子里把这三件事想清楚



1. **输入是什么？** 3 个热榜接口返回的 JSON 数据
2. **中间要怎么处理？** 把三个来源的数据统一成相同结构：标题、排名、热度、链接、图片，该有的字段对齐，没用的字段扔掉
3. **输出是什么？** 一个按日期命名的静态 HTML 文件，比如 `2026-04-01-热榜日报.html`，双击就能在浏览器里看到一份漂亮的日报页面



你不需要知道代码怎么写，但你必须知道我要从哪里到哪里，这就是需求分析



![](https://feishu.cn/file/Z6bybFmakomLuKxdj0Tcdj7un2f)



# 四、第三步：和AI一起对齐需求



到这里，你脑子里已经有了一个模糊的想法：抓 3 个热榜，生成一个 HTML 日报。但这个想法还不够具体：接口返回什么字段？页面分几个板块？文件怎么命名？



**提示词是你和 AI 聊出来的**



你可以先把模糊的想法丢给 AI：



```Bash
我想做一个热榜日报，从 36 氪、百度、抖音抓数据，生成一个静态 HTML 日报

三家的API地址是：
https://api.iyuns.com/api/hot36kr
https://api.iyuns.com/api/baiduhot
https://api.iyuns.com/api/douyinhot

你帮我想想这个事情怎么做，需要考虑哪些细节？
```



AI 会帮你补全你没想到的东西：数据字段怎么统一、文件怎么组织、页面要不要分板块、图片没有怎么处理。



你们来回聊几轮，需求就从模糊变清晰了



这个过程可以在网页端的大模型里聊（比如豆包、Kimi 的对话界面）

![](https://feishu.cn/file/AI9nbg3JMoTrx6xgJjicM7Dvnje)



最终聊完，你会得到一份像下面这样的完整提示词

**下面的内容由于不开放复制，大家可以直接截图，然后让豆包给你提取文字即可**



```Bash
帮我写一个技能，功能是生成一份每日热榜日报的静态 HTML 页面

数据来源： 请依次调用以下 3 个热榜接口获取数据：

- 36 氪热榜：https://api.iyuns.com/api/hot36kr
- 百度热搜：https://api.iyuns.com/api/baiduhot
- 抖音热榜：https://api.iyuns.com/api/douyinhot

请先调用每个接口，查看实际返回的 JSON 结构，再决定如何提取字段

数据清洗要求： 将三个来源的数据统一成相同结构，每条记录包含：榜单来源、排名、标题、热度值、原文链接、封面图（如果有）、摘要（如果有）。不同接口的字段名可能不一样，需要做映射。去掉无用字段，只保留适合展示的信息

输出要求： 生成一个纯静态 HTML 文件，文件命名格式为 YYYY-MM-DD-热榜日报.html，保存在 ./html/ 目录下。同时将清洗后的数据以 JSON 格式保存在 ./data/ 目录下，命名为 YYYY-MM-DD.json

页面设计要求： 页面是一份"日报"，不是调试页。要求：

1. 顶部显示日期、标题、数据来源说明
2. 按来源分成 3 个板块：36 氪、百度、抖音；
3. 每条内容用卡片展示，包含排名、标题、热度、摘要、链接、图片（有则显示，无则隐藏）
4. 风格清爽，信息密度适中，像一份可以发给别人看的日报
5. 页面纯静态，不依赖任何后端，本地双击即可打开
6. 兼顾桌面端和移动端的可读性我
```



还有一个心态：**不要指望 AI 一次做完**



它第一版生成的页面可能不够好看，数据字段可能映射得不对。没关系，你看了结果再告诉它哪里要改，它改了你再看这就是迭代。



**AI编程的本质不是一句话搞定，而是说你要和AI一起把事情做对**
