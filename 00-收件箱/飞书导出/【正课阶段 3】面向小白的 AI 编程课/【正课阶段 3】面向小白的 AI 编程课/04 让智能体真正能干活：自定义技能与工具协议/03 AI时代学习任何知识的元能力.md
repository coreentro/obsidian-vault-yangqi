# 写在前面



你好，我是大圣



这节课我给大家讲一个方法论：**AI时代，我们该如何学习知识**



你会看到我是如何使用Claude Code、Kimi Code这样的工具，去学会任何我想学的新东西



我管这个叫**元能力：学习能力本身**



<callout emoji="✨">
这节课如果想完整跟下来，需要解决网络问题
但并不妨碍你学习
</callout>



# 一、为什么在这里讲这一节



你可能会好奇：你一个讲 Claude Code 的课，为什么不讲功能，先讲学习方法？



因为大家接下来要跟着我的课程学东西了



但我作为一个常年写课程的人，我发现了一个现象：**在 AI 时代，教程越来越不够用了**



会有两个致命的问题



**第一：写完就过时了**



AI处在高速发展期，基本上两三天一个版本



我已经尽最大的努力去写底层逻辑，争取让它不过时，但我做不到百分之百



你看我的教程的时候，可能会发现怎么跟现在对不上了：这不是教程写得不好，是工具变得太快



**第二，教程解决不了你的「具体环境」**



每个人的系统不一样，配置不一样，网络环境不一样。



同一个操作步骤，你在 Mac 上能跑通，他在 Windows 上就报错。我写教程不可能穷举所有的情况



所以我开始意识到：**纯教程的时代已经结束了**



你更需要的不是一份步骤清单，而是一种能力：**一种不管遇到什么新工具、新知识，你都能自己搞定的能力**



这种能力在AI以前很稀缺，但AI来了之后，只要你愿意折腾，所有人都可以掌握



# 二、这个能力的底层逻辑



学习一个知识，最重要的两步是什么？



<callout emoji="✨">
**第一步：理解，**知道这个东西是什么、怎么回事
**第二步：实操，**真正动手做，在实操中反哺理解
</callout>



![](https://feishu.cn/file/WYkPb90UEoAlwdxe82rcQp3anAl)



以前只有 ChatBot 的时候，AI 能帮你解决第一步：你问它一个概念，它给你讲得很清楚



但它帮不了你实操，因为它碰不到你的电脑



**以Claude Code 为代表的产品的出现，改变了这件事**



Claude Code 是装在你电脑上的通用智能体。它不只是跟你聊天，它能在你的电脑上真实地操作，安装软件、写代码、改配置、排查 bug



所以现在的学习方法变成了这样：



你在网上看到了一篇公开的教程，可能是官方文档，也可能是别人写的教程



你把它下载下来，**你不需要自己去看**，直接丢给你的 Claude Code，告诉它：



> 这是一个 XXX 软件的安装教程，请按照里面的步骤帮我安装好



只要这个教程写得够详细，它就能帮你搞定



再进一步想：官方文档什么内容都有



如果你能找到对应的官方文档，丢给你的 Claude Code，它既能帮你总结这个页面讲了什么，也能直接帮你实操



**把 Claude Code 想象成一个万能教练：它能教你，也能替你干活**



这就是 AI 时代最强的学习方法



**而要用好这个方法，你需要两个具体的能力**



# 三、能力一：搜商



第一个能力叫**搜索能力**，我称之为「搜商」



你可能会说，不就是百度一下吗？



百度老早就有了，但为什么有的人能获取到他想要的信息，有的人获取不到？差距就在搜商



你可能还会问：我不能让 AI 帮我搜吗？



人和 AI 有各自的能力边界：如果你已经明确知道去哪个网站能找到信息，你为什么要让 AI 浪费 Token 去搜？它搜了半天可能还不是你想要的



**搜商的核心不是怎么搜，而是去哪里搜**



我给大家一个框架。当你想获取一个信息的时候，先判断它属于哪一类，然后去对应的地方找：



![](https://feishu.cn/file/VNIxbKNTsoxjVdxU50bcQtwSnN1)



**① 官方文档：**体系化学习的首选



想深入学一个工具，没有比官网更体系化的。优质的软件，它的官网文档一定是最全、最准、最及时的



**② 搜索引擎：**解决具体问题



谷歌、百度，直接搜你遇到的具体问题。比如一个报错信息，直接粘贴到搜索引擎里



**③ 优质社区：**获取实战经验



像 WayToAGI 这样的AI社区，有大量优质作者投稿，能找到很多一手的使用经验和踩坑记录



**④ 长内容平台：**学习教程和指南



B 站、YouTube、公众号，适合找系统性的教程和深度内容



**⑤ 种草平台：**获取信息差



小红书适合看别人在用什么、什么好用。但不适合深度学习



所以搜商不是打开百度就输入问题



而是脑子里先判断：这个信息，我是去官网找、去 B 站搜、去社区里翻、还是直接用搜索引擎？



**先判断去哪里，再决定怎么搜**



# 四、能力二：把内容抓下来



当你搜到了目标网页之后，下一步就是**把内容扒拉下来，变成文字**



**为什么一定是文字？**



因为 AI 最擅长处理的就是文字。你要让 Claude Code 辅助你学习和实操，就得给它文字形式的上下文



不管是网页、视频还是 PDF，最终都要变成文字交给你的 AI：



- **网页** → 下载成 Markdown 或纯文本
- **视频** → 提取逐字稿
- **PDF** → 提取文字内容



Claude Code提供了原生的Web Fetch能力，就是用来根据提供的链接获取内容



# 五、一个重要的概念：llms.txt



搜索和抓取都会了，我再给你讲一个进阶概念。这个概念会让你对整套方法论的理解更深一层



先看一个地址：https://code.claude.com/docs/llms.txt



![](https://feishu.cn/file/JPkGbV94XonIjxxTfH1cMK4wned)



这是Claude Code官方教程的导航，你会发现一团乱麻，**因为这个导航不是给人导航的，而是给AI导航的**



我拿其中一条数据给你拆解一下，你就懂了：



```Bash
[Extend Claude with skills]
(https://code.claude.com/docs/en/skills.md)
Create, manage, and share skills to extend Claude's capabilities in Claude Code. Includes custom commands and bundled skills.
```



三行信息：



- **第一行**：教程的标题：[Extend Claude with skills]
- **第二行**：这个教程对应的文档地址：https://code.claude.com/docs/en/skills.md
- **第三行**：这个教程讲什么的摘要说明



整个 llms.txt 文件就是由几十条这样的数据组成的。每一条都是：标题 + 地址 + 摘要



想象一下：**你的 Claude Code 拿到了这份文档之后，会发生什么？**



它的脑子里就有了一张完整的知识导航地图。你问它任何关于 Claude Code 的问题，它会做这几件事：



1. 在导航目录里，通过标题和摘要找到跟你问题相关的条目
2. 访问那个条目对应的 Markdown 文档地址
3. 把文档内容抓取下来，现场学习
4. 学完了，给你讲解，或者直接帮你实操



**它不是什么都知道，而是知道去哪里找**



就像你给了一个实习生一本完整的操作手册目录：他不需要背下来，遇到什么问题他知道翻哪一页就行



比如：**你问他skills是怎么用？**



![](https://feishu.cn/file/OufmbVUFEoPp1Yxg1IEc4vLXnee)



---



**这个概念为什么重要？**



因为它揭示了一个趋势：在 AI 时代，产品的官方文档不仅仅是给人看的，还必须让 AI 看得懂。



未来的学习一定是用 AI Agent 辅助学习。**如果你的文档对 Agent 不友好，用户就用不好你的产品**



越来越多的产品开始提供 `llms.txt` 这样的 AI 友好文档，比如非常火的小龙虾：https://docs.openclaw.ai/llms.txt



而你作为学习者需要知道的是：**当你想学一个新工具，先去找它有没有给 AI 准备的文档入口**



如果有，把这个入口交给你的 Claude Code，后面的事情它自己就能搞定



# 六、串起来：完整的学习流程

在正式开始学习Claude Code之前，我带大家创建一个项目，用来辅助我们接下来的学习



PS：这个方法论可以适用于你未来学习任何的知识



---



第一步：建立一个文件夹，名字叫做：claude-code-learn



第二步：在这个文件夹下进入 claude 命令行



第三步：把下面这段话复制给你的Claude Code



<callout emoji="✨">
我没有开复制权限，大家截图给豆包提取文字
</callout>



```Markdown
我要在这个文件夹下学习Claude Code

现在你帮我初始化CLAUDE.md

文档内容是：

-----

CLAUDE.md

# Claude Code 学习助手
学员正在跟着教程学习 Claude Code。你是他们的随身助手,他们遇到问题时跟你聊，你帮忙解答和解决。

# 你能帮学员做的事

- 解释概念：学员看教程遇到不懂的，用大白话讲清楚
- 提供案例：学员想动手练某个功能时，给一个能立刻试的小例子
- 排查问题：学员操作报错了，帮忙看错误信息、定位原因、给出解决办法

# 回答原则

- 说人话，别堆术语
- 一次只解决一个问题，别展开太多
- 能给例子就给例子，别光讲理论

# 知识来源

遇到问题时按这个顺序查找：

先看 refs/ 目录下已有的笔记（快、省 token）

本地没有就查官方文档：https://code.claude.com/docs/llms.txt

从官网学到有用的东西，建议存到 refs/ 里

# 关键路径

官方文档入口：https://code.claude.com/docs/en/overview.md
快速开始：https://code.claude.com/docs/en/quickstart.md
常见问题：https://code.claude.com/docs/en/troubleshooting.md
```
