# 写在前面



你好，我是大圣



前面我们通过 Kimi Code CLI，对命令行智能体有了全方位的了解



这节课我想带你看一下，全球主流的命令行智能体产品有哪些，怎么选



这节课有时效性，写于 2026 年 4 月 1 日



# 一、当前产品格局



命令行智能体这个品类，起源于 Anthropic 的 Claude Code



后来 OpenAI 的 Codex CLI、Google 的 Gemini CLI 相继上线。国内目前有 Kimi Code 和阿里的 Qwen Code



但有一个重要插曲：2026 年 3 月 31 日，Claude Code 源码意外泄露



这意味着国内很快会涌现一批命令行智能体产品，格局正在变



![](https://feishu.cn/file/PkdNbdQO3o3UooxTJIec0ASgnqg)



# 二、怎么选？看两个东西



还记得前面那张图吗？一个命令行智能体 = 执行架构（身体）+ AI 大脑



![](https://feishu.cn/file/NhuXbTpDwoGVg0x8a6OcfSlrnBd)



选产品就看这两样东西够不够强。只有强大脑没有好身体，能想到但做不到。只有好身体没有好大脑，能做但做不好



目前公认最强的身体是 Claude Code，最强的大脑是 Claude 旗舰系列



最优解当然是两者结合：但旗舰模型贵，所以实际使用中你需要按任务灵活切换大脑



![](https://feishu.cn/file/T0MvbSWnKo3n5sx6GPBcEg1Rnlf)





# 三、接下来的学习路线



接下来我会重点讲 Claude Code



命令行智能体里的核心概念：Skills、MCP等都是 Anthropic 提出的。用 Claude Code 讲最正宗



在 Claude Code 上学到的理论知识，可以无缝迁移到 Codex CLI、Kimi Code、Gemini CLI
