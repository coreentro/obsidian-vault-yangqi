# 一、为什么这节课很重要



我一直跟大家讲，龙虾就是一个智能体，而智能体最核心的部分就是它的大脑，也就是AI大模型



所以不管是现在的小龙虾，还是未来可能推出的任何智能体工具，你都必须学会如何为它灵活配置模型



这里要注意，并不是越强的模型越好



因为越强的模型越费钱。你要想让它真正为你工作，你必须在质量、金钱和速度之间做很好的取舍



**所以在未来的很长一段时间内，你要养龙虾或者养任何的智能体，核心的一件事情就是为你的工作流选择最合适的模型来工作**



配置模型的本质就是接入其他家模型的 API

# 三、给小龙虾接入模型的几种方式



在正式跟大家讲为自己的小龙虾接入模型之前，我先告诉你四种主流的方式，每一种方式都有它的优缺点



## 1）让龙虾自己配置



这个的逻辑就非常简单，你通过聊天的方式把必要的信息给到龙虾，让它自己帮你配置



它的优点很明显，你不用自己去修改任何的配置文件，你不需要了解配置文件的结构和细节  

它的缺点是，你让大模型自己做这件事，它是有可能失败的，而且如果你给的信息上下文不够，它也可能会失败



而由于你不懂配置文件的细节，就算你不断地PUA你的小龙虾，它也有一定的概率一直失败



## 2）在可视化界面配置



龙虾部署完成后，它有一个自己的可视化界面，在这个界面里可以配置模型



![](https://feishu.cn/file/SoYVbTXBGoAvE3x1Si9cxgM1nwd)



这个的优势就是，它是直接跟配置文件打交道的，没有AI介入，所以没有不确定性



但缺点是它是英文，读起来有点复杂。而且这个可视化配置界面有时候可能导致信息的缺失



---



你像 Minimax 它的配置教程就提供了前端界面修改的方式

## 3）直接修改配置文件



不管是小龙虾、Claude Code，还是你未来遇到的各种智能体工具，它都会有一个非常重要的东西，叫做配置文件



我们说到所谓的配置飞书、配置API模型、配置子智能体等等的一切，本质都是在修改这个文件



所以如果你能读懂这个配置文件的含义，并且掌握手动修改它的方法



那你才算真正读懂了龙虾，未来也才有可能真正用好龙虾
