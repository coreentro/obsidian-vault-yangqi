# 国内如何使用上CodeX

原文链接：<https://axsppz4oyvj.feishu.cn/wiki/GKHzw3DuZifd9EkZ3BZckUuvnbg?from=from_copylink>

> 说明：以下为基于原飞书文档的整理笔记。涉及模型能力、价格、风控、可用性、第三方服务等内容，均按“原文观点/原文说法”记录，未额外验证或补充来源外信息。

## 一句话概括

原文主要讨论：在国内使用 AI 编程工具时，为什么作者建议重视 CodeX，以及要真正用上 CodeX 需要解决的网络、充值、账号和手机号验证问题。

## 原文核心观点

1. Claude Code 和 CodeX 都被原文视为“执行架构”，真正影响使用体验的是背后配置的 AI 模型。
2. 原文认为，国内使用 Claude Code 的主要困难不在安装工具，而在使用 Claude 官方模型时可能面临网络、风控、支付等问题。
3. 原文推荐关注 CodeX 的核心理由是：CodeX 与 ChatGPT 账号和会员体系共享，购买 ChatGPT 官方订阅后可在 CodeX 中使用对应模型。
4. 原文称，在综合成本、额度和风控后，使用 ChatGPT 官方订阅加 CodeX 是较优先的方案之一。
5. 原文强调，用户不能长期依赖“喂饭式”教程，而要理解网络工具、账号、支付和验证等底层环节，方便后续自己排查问题。

## 背景：为什么从 Claude Code 讲到 CodeX

原文先区分了两个层面：

- Claude Code、Kimi Code、WorkBuddy、CodeX 等工具本身是执行架构。
- 真正决定工具是否好用的是所配置的模型，也就是“AI 大脑”。

因此，原文并不是简单说 CodeX 比 Claude Code 更好，而是认为在国内条件下，CodeX 可能在账号体系、订阅性价比和风控难度上更容易落地。

## Claude 官方订阅的困难

原文认为，Claude 官方订阅性价比高，但在国内使用存在明显门槛，主要包括：

- Claude 风控较严格。
- 国内访问海外网站有现实限制。
- 可能需要住宅 IP、链式代理、代充、手机接码等额外条件。
- 这个话题较敏感，也不适合不愿折腾的用户。

原文还提示，如果读者感兴趣，可以自行在 YouTube 搜索这些关键词了解原理。

## CodeX 被认为能解决的问题

原文提出 CodeX 的几个优势：

1. 账号与会员体系共享  
   原文称，购买 ChatGPT 20 美金官方订阅后，可以在 CodeX 中使用对应模型。

2. 额度体感更宽裕  
   原文称，ChatGPT 20 美金订阅在 CodeX 中的额度体感比 Claude Code 20 美金订阅更耐用。

3. 风控相对更轻  
   原文以 2026 年 5 月 6 日为时间点，认为使用 GPT 和 CodeX 相比 Claude 官方订阅，风控程度较低。

4. 可使用图像模型  
   原文提到 GPT 的 Image 2.0，并认为它是很强的图像模型，且可以在 CodeX 中使用。

## 原文给出的四个优先级

在综合考虑成本的前提下，原文给出的优先级是：

1. GPT 20 美金官方订阅 + CodeX。
2. Claude Code + 国内模型，例如 GLM5.1 或 MinMax2.7。
3. Claude Code + Claude 20 美金官方订阅。
4. Claude Code + Claude 中转 API。

原文补充：如果用量大，并且能解决网络问题，Claude Code 的 100 美金或 200 美金官方订阅也可能划算。这个优先级不一定适用于每个人，用户应按自己的需要选择。

## 使用 CodeX 需要解决的四个问题

### 1. 网络问题

原文把网络问题称为敏感话题，让读者自行阅读另一个教程：

- 飞书引用文档：第三节：方案一·买一个现成的网络工具
- 引用中的 doc-id：`NAKhwOoW3imlOHkEP9Wcsi3inJh`

原文还提示：该部分没有讲住宅 IP，后续使用 Claude 官方订阅时可能需要更换工具；如果想自己先一步到位，可以搜索住宅 IP、链式代理、代充、手机接码等关键词。

### 2. 充值问题

原文指出，海外工具不能使用国内银行卡充值，通常需要海外信用卡，由此产生代充服务。

原文提到的代充方式包括：

- 苹果礼品卡。
- 利用网站漏洞的低价代充。
- 闲鱼上的拼车、代充服务。
- 作者朋友使用过的某个代充渠道。

原文同时明确声明：

- 不为推荐对象背书。
- 不抽佣金。
- 没有要折扣价。
- 不保证最便宜。
- 价格大致与官方价接近，约 140 RMB 左右。

### 3. 账号问题

原文认为，要使用 CodeX 和 GPT，必须有账号。

可选路径包括：

- 有自己的账号，则可以找别人代充。
- 如果搞不定自己的账号，可以购买成品号。
- 购买成品号时，具体价格需要自行询问。

### 4. 手机接码

原文称，CodeX 在 2026 年 4 月下旬需要验证手机号，并且不能使用国内手机号，因此可能需要海外手机号接码平台。

原文列出的接码平台：

- <https://hero-sms.com/cn>
- <https://lubansms.com/v2/receiveSms>
- <https://www.textverified.com/app/verifications>

原文提醒：这些平台作者自己没有用过，是朋友推荐；如果接不到验证码，作者也无法保证解决。

## 作者想强调的能力

原文后半部分的重点，不只是教读者使用某个工具，而是强调：

- 这类工具和服务环境会变化，单靠一次性教程可能很快失效。
- 用户需要理解网络、账号、支付、验证等基础逻辑。
- 具备最基础的网络工具后，可以在 YouTube 上主动搜索问题。
- 新手应完整阅读相关网络工具教程的视频和文字。

从实践角度看，原文更像是在训练读者建立“自己排查问题”的能力，而不只是照步骤安装。

## 写在最后

原文结论是：Claude Code、CodeX 这类 AI 编程工具应当被战略性重视。

原文建议：

- 条件允许时，至少使用 GPT5.5 或 Claude Opus 模型中的一个。
- 条件不允许时，也可以使用 Claude Code + GLM5.1、MinMax2.7、DeepSeek V4 等组合。
- WorkBuddy、KimiCode 等工具也应开始使用，即使它们还不完美。
